# Przychodnia
Aplikacja dla przychodni
Aplikacja internetowa umożliwia elektroniczną rejestrację wizyt przez klientów w klinice weterynaryjnej. Ma to być aplikacja dla kliniki weterynaryjnej i jej klientów na potrzeby takie, jak umawianie wizyt swoich zwierząt przez klientów do weterynarzy w klinice weterynaryjnej i dostęp do listy tych wizyt przez lekarzy przez co wiedzą oni o terminach swoich wizyt i ich szczegółach. Umożliwia to wszystko elektroniczny proces rejestracji klientów do poszczególnych weterynarzy. W pracy wykorzystano wiele technologii webowych, ze szczególnym naciskiem na Spring Boot. Wykorzystano m.in. technologie takie jak Java i jej framework Spring Boot po stronie serwera aplikacji. Do komunikacji aplikacji z bazą danych wykorzystano takie narzędzia jak JPA i Hibernate do odwzorowania modelu ORM, JDBC do komunikacji z bazą danych za pomocą SQL i wykorzystano również zorientowany obiektowo język JPQL do wykonywania zapytań niskiego poziomu do relacyjnej bazy danych. Użyto również relacyjnego systemu bazodanowego PostgreSQL. Wykorzystano również  technologie takie jak m.in. HTML, CSS, Bootstrap i JavaScript po stronie klienta. Wykorzystano również Thymeleaf, który jest silnikiem szablonów Javy dla HTML5, ułatwiającym tworzenie aplikacji po stronie klienta, generując zawartość stron (widoków HTML) głównie po stronie serwera. Do zarządzania zależnościami w projekcie wykorzystano narzędzie automatyzujące budowę oprogramowania na platformę Java Apache Maven. Technologia Spring Web MVC pozwaliła na napisanie aplikacji webowej w modelu MVC. Architektura aplikacji jest monolityczna. Wybrane do projektu IDE to IntelliJ IDEA.
